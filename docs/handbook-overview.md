# Handbook Overview — Good Neighbor Protocol

This document describes how the GNP handbook is organized and how to navigate it.

---

## What Is the Handbook?

The GNP handbook is the living documentation of the Good Neighbor Protocol framework. It is not a one-time document — it is intended to grow alongside the framework as it is tested, refined, and used in real contexts.

The handbook lives in this repository. The most authoritative document in it is [source-of-truth.md](source-of-truth.md).

## How to Use This Handbook

**If you're new to GNP:**
Start with the [README](../README.md), then read [source-of-truth.md](source-of-truth.md) to understand the framework in full.

**If you want to apply GNP to a real request:**
Go to the [CASSEROLE Checklist](casserole-checklist.md) and work through it with your request.

**If you have questions:**
Check the [FAQ](faq.md) first. If your question isn't there, open a GitHub Discussion.

**If you want to understand where this is heading:**
Read the [Roadmap](roadmap.md).

---

## A Note on Scope

This handbook is deliberately conservative. It documents the framework as it exists today — not a future vision, not a pitch, not a product spec. If something isn't here, it hasn't been built yet.

The [source-of-truth.md](source-of-truth.md) file is the reference point for any disagreement between documents. When in doubt, that file takes precedence.

---

## Updating the Handbook

Changes to the framework's core concepts should be reflected in [source-of-truth.md](source-of-truth.md) first. Changes to how the framework is applied or explained should be reflected in the relevant handbook document.

If you are contributing to this project, open a Discussion before making significant changes to core documents.
